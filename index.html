Note: Like Bootstrap, this implementation uses data-* attributes. However, they have been changed such that the -bs- namespace has been removed. Thus, it will not work directly with the associated Bootstrap 5 data attributes unless one modifies the code in the Carousel constructor. This also means that if you intend to use the default styles that you have to add the following selectors:


  .carousel-indicators [data-target] {
    box-sizing: content-box;
    flex: 0 1 auto;
    width: 30px;
    height: 3px;
    padding: 0;
    margin-right: 3px;
    margin-left: 3px;
    text-indent: -999px;
    cursor: pointer;
    background-color: #fff;
    background-clip: padding-box;
    border: 0;
    border-top: 10px solid transparent;
    border-bottom: 10px solid transparent;
    opacity: 0.5;
    transition: opacity 0.6s ease;
  }

  @media (prefers-reduced-motion: reduce) {
    .carousel-indicators [data-target] {
      transition: none;
    }
  }

  .carousel-dark .carousel-indicators [data-target] {
    background-color: #000;
  }

  /* rewrite */
  .carousel-indicators .active {
    opacity: 1;
  }

Bootstrap 5 provides the .ratio helper for creating aspect-ratio containers. It was used as follows:


  <div class="carousel-inner ratio ratio-16x9">

However, to get it to work correctly with the <img>s in the current implementation, the following CSS rule was added:


  .carousel-inner.ratio img {
    display:block;
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    max-width: 100%;
    max-height: 100%;
  }

Note: With this rule, images will extend to either 100% of the width or height of the parent .carousel-item. However, the trade-off is that they will not stretch beyond their default resolution. This means that the images have to be sufficiently large enough to begin with. Alternatively, one could write a javascript function that checks width against height, and conditionally sets either width:100%, or height:100%. That extra step has been omitted in the current implementation.

The goal for this demo was to create a working carousel using bootstrap.css. However, it's likely that if this code was ever used it would actually be without bootstrap.js. Consequently, it doesn't make sense to use the -bs- namespace. Also, it's important to note that this project was built with bootstrap@5.0.0-beta2. Currently, the CSS styles are built on top of that stylesheet. However, if you wanted to make it completely independent of Bootstrap you would need to abstract the carousel-related CSS from bootstrap.css
