Note: Like Bootstrap, this implementation uses data-* attributes. However, they have been changed such that the -bs- namespace has been removed. Thus, it will not work directly with Bootstrap data attribute unless you tweak the code in Carousel.js. This also means that if you intend to use the default styles that you have to add the following selectors:


  .carousel-indicators [data-target] {
    box-sizing: content-box;
    flex: 0 1 auto;
    width: 30px;
    height: 3px;
    padding: 0;
    margin-right: 3px;
    margin-left: 3px;
    text-indent: -999px;
    cursor: pointer;
    background-color: #fff;
    background-clip: padding-box;
    border: 0;
    border-top: 10px solid transparent;
    border-bottom: 10px solid transparent;
    opacity: 0.5;
    transition: opacity 0.6s ease;
  }

  @media (prefers-reduced-motion: reduce) {
    .carousel-indicators [data-target] {
      transition: none;
    }
  }

  .carousel-dark .carousel-indicators [data-target] {
    background-color: #000;
  }

  /* rewrite */
  .carousel-indicators .active {
    opacity: 1;
  }

The goal for this demo was to create a working carousel using bootstrap.css. However, it's likely that if this code was ever used it would actually be without bootstrap.js. Consequently, it doesn't make sense to use the -bs- namespace. Also, it's important to note that this project was built with bootstrap@5.0.0-beta2. Currently, the CSS styles are built on top of that stylesheet. However, if you wanted to make it completely independent of Bootstrap you would need to abstract the carousel-related CSS from bootstrap.css
